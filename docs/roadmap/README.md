## Organization

Features are numbered by priority:
- **001-**** - High priority, near-term implementation
- **002-**** - Medium priority, depends on prerequisites
- **003-**** - Long-term features

## How to Propose a New Feature

1. Create a new markdown file with prefix `00X-feature-name.md`
2. Include:
   - Clear problem statement
   - Proposed solution
   - Implementation details
   - Effort estimate
   - Dependencies
3. Reference it in the main [future-advanced-features.md](future-advanced-features.md) document

## Status Legend

- **Proposed**: Idea documented, not yet approved
- **Approved**: Ready for implementation
- **In Progress**: Development underway
- **Completed**: Implemented and released
